# Network Troubleshooting Lab

## Overview
This project documents my process for diagnosing and resolving common network connectivity issues in an IT support environment.

## Objectives
- Troubleshoot network connectivity problems
- Identify common causes of network failures
- Use diagnostic tools to analyze network issues
- Document troubleshooting steps and resolutions

## Scenario
A user reports that they are unable to access company resources and experience intermittent network connectivity issues.

## Troubleshooting Process

### 1. Gather Information
- Asked the user questions to identify symptoms
- Determined whether the issue affected one user or multiple users
- Verified error messages and affected applications

### 2. Physical Checks
- Confirmed network cable connections
- Verified device power status
- Checked network adapter status

### 3. Diagnostic Commands Used

**ipconfig**
- Verified IP address configuration
- Checked default gateway and DNS settings

**ping**
- Tested connectivity between devices

**tracert**
- Identified potential routing issues

**nslookup**
- Verified DNS resolution

## Example Resolution

Issue:
User could not access network resources.

Cause:
Incorrect IP configuration.

Resolution:
- Renewed IP address
- Verified DNS settings
- Confirmed successful network connectivity
